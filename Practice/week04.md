# 4주차 실습 정리

## 오늘 한 것

* 원형 / AABB / OBB Bounding Box 시각화
* 세 방식의 충돌 판정 차이 관찰

## 관찰 내용

### 원형 vs AABB

원형 충돌은 중심 사이의 거리를 기준으로 계산되기 때문에 실제 오브젝트의 움직임과 자연스럽게 맞는 충돌 판정을 보여주었다 특히 대각선 방향에서 접근할 때도 부드럽고 정확하게 충돌을 감지하는 것이 특징이었다

반면 AABB는 축에 정렬된 사각형 기준으로 계산되기 때문에 연산이 매우 빠르지만 실제로 닿지 않았는데도 충돌로 판정되는 경우가 있었다 즉 성능은 좋지만 정확도는 떨어질 수 있다는 것을 직접 확인했다

### AABB vs OBB

AABB는 항상 화면 축에 맞춰진 사각형이기 때문에 오브젝트가 회전해도 그 회전을 반영하지 못한다 그래서 회전된 상태에서는 충돌 판정이 부정확해지는 문제가 발생한다

반면 OBB는 오브젝트의 회전까지 반영한 사각형 충돌 방식이다 따라서 실제 오브젝트 모양과 훨씬 더 일치하는 정확한 충돌 판정을 제공한다 하지만 SAT 알고리즘을 사용하기 때문에 계산이 복잡하고 성능 부담이 커질 수 있다는 점도 함께 느낄 수 있었다

## AI와의 대화에서 배운 것

이번 실습에서 나는 충돌 방식마다 왜 결과가 다르게 나오는지와 회전된 오브젝트는 어떻게 충돌을 계산하는지를 질문했다

AI의 설명을 통해 다음을 이해했다

AABB는 가장 단순하고 빠른 충돌 방식이지만 회전을 반영하지 못한다
원형 충돌은 중심 거리 기반으로 계산되어 더 자연스러운 충돌 판정을 제공한다
OBB는 회전된 사각형 충돌 방식이며 SAT(분리축 정리)를 이용해 충돌을 판정한다
특히 SAT 알고리즘은 물체를 특정 축에 투영했을 때 겹치는지를 기준으로 충돌을 판단한다는 점이 인상적이었다.

또한 AI를 통해 충돌 판정에는 항상 정확도 vs 성능의 트레이드오프가 존재하며 실제 게임에서는 하나의 방식만 사용하는 것이 아니라 상황에 따라 적절히 혼합해서 사용하는 것이 중요하다는 것을 배우게 되었다.

## 내 게임에 적용한다면

내 게임에서는 모든 오브젝트에 동일한 충돌 방식을 사용하는 것이 아니라 상황에 따라 나누어 사용할 것이다

일반 오브젝트(배경 오브젝트 장애물 등) :AABB 사용 → 빠른 연산으로 성능 유지
플레이어 및 주요 적: 원형 또는 OBB 사용 → 더 정확한 충돌 판정 필요

이렇게 하면 전체적인 성능을 유지하면서도 플레이어가 체감하는 충돌 정확도는 높일 수 있다 따라서 성능과 정확도를 균형 있게 고려한 방식으로 구현할 것이다

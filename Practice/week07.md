# Week 3 실습 기록

## 목표
Pygame을 활용하여 블럭 깨기 게임 기능 확장 및 사운드 시스템 구현

## AI 대화 기록

### Q1: Pygame에서 배경 음악(BGM)이 재생되지 않는 이유는?

- AI 답변: 파일 경로 또는 파일 형식 문제일 가능성이 높음
- 시행착오:
  - mp3 파일 → 손상된 파일로 재생 실패
  - wav 파일 → 지원되지 않는 형식 (Unknown WAVE format)
  - ogg 파일 → 잘못된 변환 (VORBIS 오류)
- 최종 해결:
  - Audacity를 사용하여 OGG Vorbis 형식으로 다시 변환
  - assets/music 폴더에 넣어 정상 재생 성공

### Q2: Pygame에서 효과음(히트 사운드)을 넣는 방법은?

- AI 답변: pygame.mixer.Sound() 사용
- 적용 코드:hit_sound = pygame.mixer.Sound("assets/music/hit.wav")
hit_sound.play()
- 적용 결과: 블럭 충돌 시 효과음 정상 재생

### Q3: 블럭이 깨질 때마다 소리를 재생하려면 어디에 코드를 넣어야 하나?

- AI 답변: 충돌 처리 코드 내부 (if hit_block: 부분)에 넣어야 함
- 시행착오:
  - 위치를 잘못 넣어서 소리가 안 나옴
- 최종 해결:
  - 충돌 감지 직후에 hit_sound.play() 추가
- 적용 결과: 블럭 맞을 때마다 효과음 재생 성공

### Q4: 사운드 파일이 있어도 재생이 안 되는 이유는?

- AI 답변: 파일 형식이 pygame에서 지원되지 않을 수 있음
- 시행착오:
  - 확장자만 변경했지만 실제 형식이 달라 오류 발생
- 최종 해결:
  - 반드시 오디오 프로그램으로 정식 변환(export) 해야 함

## 느낀 점

이번 실습을 통해 단순히 코드를 작성하는 것뿐만 아니라  
파일 경로, 파일 형식, 라이브러리 지원 여부까지 고려해야 한다는 점을 알게 되었다.

특히 사운드 관련 오류를 해결하면서  
문제의 원인을 로그를 통해 분석하는 것이 중요하다는 것을 배웠다.

또한 AI의 도움을 받아 빠르게 해결할 수 있었지만,  
단순히 따라하는 것이 아니라 왜 그런지 이해하는 과정이 중요하다고 느꼈다.

## 앞으로의 개선 방향

- 사운드뿐만 아니라 다양한 효과 (폭발, UI 효과음 등) 추가
- 코드 구조를 더 정리하여 유지보수 쉽게 만들기
- 에러 로그를 스스로 분석하는 능력 키우기
- AI 도움 없이도 기본적인 문제 해결 능력 향상

# 6주차 실습 노트

## 1. 실습 목표 및 느낀 점  
이번 실습에서는 스프라이트와 사운드의 기초를 배우고, 이를 직접 코드에 적용해보는 과정을 진행했다. 특히 단순히 코드를 실행하는 것이 아니라, 주석을 해제하고 값을 변경하면서 결과가 어떻게 달라지는지를 확인하는 과정이 인상 깊었다. 이를 통해 게임에서 시각적 요소와 청각적 요소가 얼마나 중요한 역할을 하는지 체감할 수 있었다.  

또한 스프라이트 시트 도구를 활용하여 애니메이션을 생성하는 과정은 처음에는 생소했지만, 자동으로 코드가 생성되는 점에서 매우 효율적이라고 느꼈다. 앞으로 게임을 개발할 때 애니메이션 구현 시간을 크게 줄일 수 있을 것 같다.

## 2. 스프라이트 실습 내용  
스프라이트 실습에서는 이미지의 크기 조절, 위치 변경, 회전 등을 직접 적용해보았다.  

특히 이미지의 크기를 변경하거나 회전할 때 **적용 순서에 따라 결과가 달라지는 점**이 흥미로웠다. 예를 들어 확대 후 회전을 할 때와 회전 후 확대를 할 때 결과 이미지가 다르게 나타났는데, 이는 게임 개발에서 처리 순서가 매우 중요하다는 것을 보여주는 부분이라고 생각한다.  

또한 Rect 속성을 이용해 이미지의 위치를 조정하는 과정에서, 객체의 기준점(anchor)이 어떻게 작용하는지도 이해할 수 있었다.

## 3. 사운드 실습 내용  
사운드 실습에서는 효과음과 배경음악을 적용하고, 재생 방식과 볼륨을 조절해보았다.  

특히 배경음악을 반복 재생하거나, 특정 키 입력 시 효과음을 재생하는 기능을 구현하면서 **게임의 몰입도를 높이는 요소가 사운드라는 점**을 느꼈다.  

또한 볼륨 값을 조절하면서 미세한 차이도 사용자 경험에 영향을 준다는 것을 알게 되었고, 단순히 소리를 넣는 것이 아니라 적절한 설정이 중요하다는 것을 이해하게 되었다.

## 4. 스프라이트 시트 및 애니메이션  
스프라이트 시트를 활용하여 애니메이션을 구현하는 과정은 이번 실습에서 가장 핵심적인 부분이었다.  

여러 프레임을 순서대로 배치하여 캐릭터의 움직임을 자연스럽게 표현할 수 있었고, 프레임 속도를 조절함으로써 애니메이션의 빠르기를 변경할 수 있었다.  

특히 같은 프레임을 반복하거나 순서를 바꾸는 것만으로도 다양한 움직임을 만들 수 있다는 점에서, 적은 리소스로도 다양한 연출이 가능하다는 것을 배웠다.

## 5. AI 활용 경험  
이번 실습에서는 AI를 활용하여 코드 작성 및 수정에 도움을 받았다.  

예를 들어 이미지 이동, 사운드 재생 방식 변경 등의 기능을 구현할 때 AI에게 질문을 하여 코드를 생성했고, 이를 직접 실행하며 이해하는 과정을 거쳤다.  

AI를 활용하면서 느낀 점은, 단순히 코드를 받아 쓰는 것이 아니라 **왜 그렇게 동작하는지 이해하는 것이 중요하다**는 것이었다. 일부 코드는 그대로 사용하기 어려워 수정이 필요했고, 이 과정에서 코드 이해도가 높아졌다.

## 6. 적용 결과  

### ✔ 잘 된 점  
- 스프라이트와 사운드를 게임에 정상적으로 적용할 수 있었다  
- 애니메이션을 구현하여 캐릭터 움직임을 표현할 수 있었다  
- AI를 활용해 빠르게 기능을 구현할 수 있었다  

### ❗ 어려웠던 점  
- 스프라이트 시트 프레임 설정이 처음에는 헷갈렸다  
- 애니메이션 속도 조절과 타이밍 맞추기가 어려웠다  
- 코드 수정 과정에서 오류가 발생했을 때 원인을 찾는 것이 힘들었다  

## 7. 앞으로 개선할 점  
앞으로는 단순히 기능을 구현하는 것에서 끝나는 것이 아니라, 코드 구조를 더 깔끔하게 정리하고 재사용성을 고려해야겠다고 생각했다.  

또한 애니메이션 상태를 분리하여 걷기, 점프, 공격 등의 다양한 상황을 표현할 수 있도록 확장해보고 싶다.  

마지막으로 사운드와 애니메이션을 더욱 자연스럽게 연결하여 게임의 완성도를 높이는 방향으로 발전시키고자 한다.

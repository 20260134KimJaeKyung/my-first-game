## 오늘 한 것

### 1. BSP Dungeon 생성

bsp_dungeon.py를 실행하여 BSP(Binary Space Partitioning) 알고리즘으로 던전이 생성되는 과정을 확인하였다.

확인한 내용

* 공간이 여러 구역으로 분할되는 과정
* 분할된 공간에 방이 생성되는 과정
* 방과 방을 연결하는 통로 생성 과정
* MST(Minimum Spanning Tree)를 이용한 통로 정리 과정
* Seed 값에 따라 다른 던전이 생성되는 모습

---

### 2. Cellular Automata 동굴 생성

cellular_cave.py를 실행하여 Cellular Automata 알고리즘으로 동굴이 생성되는 과정을 확인하였다.

확인한 내용

* 초기 랜덤 맵 생성
* 반복 연산을 통해 자연스러운 동굴 형태로 변화
* FILL_PROB 값을 변경하여 벽의 밀도 변화 확인
* ITERATIONS 값을 변경하여 동굴 형태 변화 확인

---

### 3. Perlin Noise 지형 생성

noise 라이브러리를 설치한 후 perlin_terrain.py를 실행하였다.

```bash
pip install noise
```

확인한 내용

* Perlin Noise를 이용한 자연스러운 지형 생성
* Scale 값에 따른 지형 변화
* OCTAVES 값에 따른 세부 지형 변화
* Seed 값에 따라 다른 지형 생성

## 가장 흥미로웠던 실습

가장 흥미로웠던 실습은 Perlin Noise 지형 생성이었다.

랜덤으로 생성된 것처럼 보이지만 자연스럽게 이어지는 지형이 만들어지는 점이 인상적이었다. 특히 Scale 값을 조정할 때 지형의 형태가 크게 달라지는 것을 확인할 수 있었고, 실제 게임의 월드 생성에도 활용될 수 있다는 점이 흥미로웠다.

---

## 파라미터를 변경했을 때의 변화

### BSP Dungeon

* Seed 값을 변경하면 던전 구조가 완전히 달라졌다.
* 방의 위치와 통로 연결 방식이 매번 다르게 생성되었다.

### Cellular Automata

* FILL_PROB 값을 높이면 벽이 많아져 좁고 복잡한 동굴이 생성되었다.
* FILL_PROB 값을 낮추면 넓은 공간이 많이 생성되었다.
* ITERATIONS 값을 늘릴수록 동굴 형태가 더욱 자연스럽게 정리되었다.

### Perlin Noise

* Scale 값을 낮추면 부드럽고 완만한 지형이 생성되었다.
* Scale 값을 높이면 울퉁불퉁하고 험준한 지형이 생성되었다.
* OCTAVES 값을 높이면 세부적인 지형 표현이 증가하였다.

## 기말 프로젝트 활용 가능성

프로시저럴 생성은 게임 개발에서 매우 유용하게 활용될 수 있다고 생각한다.

특히 내가 만들고 싶은 게임에서도 활용할 수 있을 것 같다.

* BSP를 이용한 랜덤 던전 생성
* Perlin Noise를 이용한 섬 또는 지형 생성
* Cellular Automata를 이용한 동굴 생성
* Seed를 활용한 재현 가능한 맵 생성

이를 통해 플레이할 때마다 새로운 환경을 제공할 수 있으며 게임의 반복 플레이 재미를 높일 수 있을 것이라고 생각한다.

## 느낀 점

이번 실습을 통해 프로시저럴 생성이 단순한 랜덤 생성이 아니라 일정한 규칙과 알고리즘을 기반으로 동작한다는 것을 알게 되었다. 특히 Seed 값을 사용하면 같은 결과를 다시 생성할 수 있다는 점이 신기했다. 또한 게임 개발에서 랜덤 맵 생성, 지형 생성, 동굴 생성 등에 활용할 수 있다는 점을 이해할 수 있었으며 앞으로 개인 프로젝트에도 적용해보고 싶다.
